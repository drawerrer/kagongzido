# 카공지도 프로젝트 가이드

## 커밋 메시지 규칙

GitHub에 작업 내용을 업로드할 때 반드시 아래 규칙에 맞춰 커밋 메시지를 작성한다.

| 태그 | 사용 상황 | 예시 |
|------|-----------|------|
| `feat:` | 새 기능 추가 | `feat: 즐겨찾기 추가 기능 구현` |
| `fix:` | 버그 수정 | `fix: 위치 권한 거부 시 앱 크래시 수정` |
| `style:` | UI/디자인 수정 | `style: 필터 버튼 색상 변경` |
| `chore:` | 설정·패키지 변경 | `chore: ESLint 설정 추가` |
| `refactor:` | 코드 구조 개선 | `refactor: 지도 컴포넌트 분리` |
| `docs:` | 문서 작성·수정 | `docs: README 업데이트` |
